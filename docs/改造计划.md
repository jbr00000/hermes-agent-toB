# Hermes Agent → 私有 to-B 智能体框架 · 改造计划

> 本文档是把 Hermes Agent 开源框架改造成面向 to-B 场景的私有智能体框架的总纲。
> 所有结论来自一次完整的设计 grilling，9 个架构决策已锁定。

---

## 需求描述（要建什么 & 为什么）

### 1. 项目定位
在开源智能体框架 Hermes Agent 基础上，硬分叉改造成**面向 to-B 场景的私有智能体框架**。不是个人助手，而是企业级、可交付给客户独立部署的生产力 agent 平台。

### 2. 目标用户与使用场景
- **目标客户**：中小企业（中小客户）。
- **单客户规模**：每个客户十几到几十人使用。
- **使用关系**：同一客户内的多个用户**共享一个部署**，但**人与人之间的会话必须隔离**，且需**单独管理**（身份、权限、审计各自独立）。
- **客户数量级**：先小规模起步（几十个客户），架构上预留扩展到上百甚至上千的空间。

### 3. 功能性需求（必须支持）

| # | 需求 | 说明 |
|---|---|---|
| F1 | **自生成代码 + 沙盒执行** | agent 能写代码并在隔离沙盒里运行，完成数据分析、批处理等复杂任务 |
| F2 | **数据库读权限开放** | agent 能读客户数据库做查询、分析、问答（如 nl2sql） |
| F3 | **自有能力自由调用** | 我们自研的智能体、算法、数据接口要能被 agent 自由调用 |
| F4 | **sub-agent（子智能体）** | 主 agent 能派生子 agent 做并行/隔离的子任务 |
| F5 | **MCP** | 作为 MCP 客户端连接外部 MCP server，按需扩展工具能力 |
| F6 | **内置 todo** | agent 内置任务规划与跟踪 |
| F7 | **plan mode（规划模式）** | 先调研产出方案、经用户批准后再执行 |
| F8 | **多智能体协同** | 多个 agent 协作完成工作流 |
| F9 | **持久记忆 + 自进化 skill + 跨会话沉淀** | 跨会话累积知识与技能，skill 能自我改进（后续会做改造，但地基要留） |
| F10 | **自有前端界面** | 框架提供后端 API，前端由我们自己建设 |

### 4. 非功能性需求（硬约束）

- **S1 安全性 / 隐私性（最高优先级）**：
  - 客户数据**不得离开客户环境**（跨客户物理隔离）。
  - agent **不得拥有操作宿主服务器的权限**（不能跑 shell、不能改主机文件）。
  - 同一客户内**用户间数据不得串**（记忆、会话、审计严格按人隔离）。
  - 数据库**只读**必须是不可绕过的强制（连沙盒里生成的代码也写不了）。
- **S2 可控的成本结构**：自托管开模型为主，云模型按需 opt-in，避免被单一云厂商 token 计费绑定。
- **S3 可运维**：单租户独立部署，先支持几十客户半自动运维，未来可平滑扩容。

### 5. 明确不做（Out of Scope）

- ❌ **computer use**（控制鼠标键盘操作桌面）——不开放这类能力。
- ❌ **TUI**（终端 UI）——我们有自己的 Web 前端。
- ❌ **消息平台网关**（Telegram/Discord/Slack 等 20 个平台）——非 to-B 内部工具的渠道。
- ❌ **公共 skill 市场 / 云记忆服务**——数据外发，违反隐私。

### 6. 部署与商业约束

- **部署形态**：单租户、每客户独立部署（客户机房 on-prem 或专属 VPC）。
- **模型来源**：两类客户——① 自托管开源模型（OpenAI 协议部署）；② 国产云模型（智谱 GLM、阿里云百炼 Qwen、DeepSeek）。
- **扩展策略**：先按几十客户落地；每个客户部署**完全声明式 + 容器化**，未来扩到上百客户不用重写代码。

## 1. 执行原则（适用于 step 1 / step 2）

1. **先 git init + 基线 commit**：当前目录不是 git 仓库。第一步 `git init` → 初始 commit（原始快照）→ 之后每个子步骤一个 commit，便于回退。
2. **建立 `UPSTREAM.md`**：记录 fork 自 NousResearch/hermes-agent 的 commit SHA + 日期。
3. **软禁用 vs 硬删除**：
   - 硬删除 = 确定不要（删文件）。
   - 软禁用 = 可能复用（保留文件，取消注册 / 不导入 / 从 toolset 摘掉）。
4. **迭代式删除**：删一批 → `scripts/run_tests.sh` → 修 import 错误 → 重复。**绝不一次删完才测**。
5. **保留的核心文件绝不误删**：`run_agent.py` / `model_tools.py` / `toolsets.py` / `hermes_state.py` / `hermes_constants.py` / `hermes_cli/config.py` / `agent/`（memory_manager + curator + context + auxiliary）/ `tools/registry.py`。

## Step 3：自建与改造

> 这是真正「改造成你的框架」的部分。按依赖顺序排列。

### 3.1 基础设施先行
- [ ] **3.1.1** `git init` + 初始基线 commit + `UPSTREAM.md`（决策 1）。
- [ ] **3.1.2** 声明式部署配置：每客户一份 `deployment.yaml`（启用的 MCP server / skill / 模型选择 / DB 凭证引用）。代码不写死任何客户细节（策略）。
- [ ] **3.1.3** 容器化：agent + 沙盒容器 + 本地 skill-hub + MCP server 全部打容器，docker-compose（未来平滑升 k8s，策略）。

### 3.2 核心：headless 服务 + 多用户（决策 2/3/4）
- [ ] **3.2.1** 新建 headless FastAPI server（你前端的 BFF）：登录、按人会话、SSE/WebSocket 流式、工具审批 HTTP 化。直接对接 `AIAgent`(run_agent.py) + `SessionDB`(hermes_state.py)。
- [ ] **3.2.2** 多用户体系：身份认证（登入）、用户管理（管理员增删人）、**按 user_id 隔离会话/记忆/skill 调用/审计**、角色权限（管理员 vs 普通用户）。
- [ ] **3.2.3** 审计日志：谁在何时调用了什么工具、查了什么数据。

### 3.3 能力面（决策 5/6）
- [ ] **3.3.1** `db_query` 核心工具：连客户的库，**只读凭证在 DB GRANT 层强制**（工具和沙盒共用同一份只读凭证）。
- [ ] **3.3.2** PTC 沙盒白名单收死：`{db_query, 你的 MCP 工具}`，其余全摘掉。
- [ ] **3.3.3** 沙盒收紧：无宿主 bind-mount、网络出口白名单（只放行 DB + 你的服务端点）、只读 DB 凭证按会话注入、容器池预热（性能）。
- [ ] **3.3.4** 你的算法/数据接口 → 包装成 MCP server（按决策 6 规则，逐个判断核心工具 vs MCP）。
- [ ] **3.3.5** 专门 agent（如 nl2sql 行业问数）→ MCP 包裹的 agent 服务（对主 agent 是黑盒工具）。

### 3.4 记忆 / skill / 跨会话（决策 7/7b）
- [ ] **3.4.1** 本地 `MemoryProvider`（实现 ABC）：数据存客户自己的 DB/SQLite，**按 user_id 隔离**（扩 ABC 的 prefetch/sync_turn 带 user_id）。
- [ ] **3.4.2** 跨会话沉淀：基于 SessionDB FTS5，按 user_id 范围检索历史会话。
- [ ] **3.4.3** 本地私有 skill-hub 服务：企业用户上传/管理 skill + **审批工作流**（管理员审批才上线）+ 共享/私有分层 + 复用 guard/audit/provenance。

### 3.5 plan mode（决策 9）
- [ ] **3.5.1** 会话级 plan/execute 模式标志 + toolset 门控（plan 模式只读工具集，禁变更类）。
- [ ] **3.5.2** 审批闸门：用户批准方案 → 切 execute → 恢复完整工具集。

### 3.6 运维（策略）
- [ ] **3.6.1** 部署/升级脚本（先支持几十个客户手工 + 半自动）。
- [ ] **3.6.2** 监控 + 配额（每个客户部署的资源/Token 配额）。
- [ ] **3.6.3** 留好「上百客户」的控制面扩展口（声明式配置 + 容器编排已铺好基础）。

- 🔴 **绝不在 step 1 误删**：`run_agent.py` / `model_tools.py` / `toolsets.py` / `hermes_state.py` / `agent/`(尤其 memory_manager + curator + context + auxiliary) / `tools/registry.py`。
